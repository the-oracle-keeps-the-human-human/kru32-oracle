# CI/CD and Release Process Analysis

## CI Pipeline: Pull Request Validation

Workflow: `.github/workflows/pull-request-ci.yml`
- Trigger: All pull requests
- Runner: ubuntu-latest, 10-minute timeout
- Node.js 22 with npm cache

### Steps
1. Checkout (pinned action for supply-chain security)
2. Setup Node.js 22
3. `npm ci` (clean install)
4. Install Codex CLI globally
5. Run test suite (`node --test`)
6. Build (type-check via tsc)

## Version Management

### bump-version.mjs Script

Maintains version consistency across 5 locations in 4 files:
1. `package.json` version
2. `package-lock.json` (root + nested)
3. `plugins/codex/.claude-plugin/plugin.json`
4. `.claude-plugin/marketplace.json` (metadata + plugins array)

### Operations
- `node scripts/bump-version.mjs <version>` -- update all manifests
- `node scripts/bump-version.mjs --check` -- verify consistency without modifying

### Semver Validation
Allows standard semver plus pre-release and build metadata suffixes.

## Release Process

**Manual** -- no automated release workflow. Steps:
1. Update CHANGELOG.md
2. Bump version via script
3. Commit, PR, CI validates
4. Merge to main
5. Create GitHub Release (manual)

## What CI Checks

| Check | Tool | Purpose |
|-------|------|---------|
| Type Safety | TypeScript | Catch type errors |
| Unit Tests | node --test | Verify behavior |
| Codex Integration | codex app-server | Validate generated types |
| Runtime Build | TypeScript prebuild | Verify execution |
