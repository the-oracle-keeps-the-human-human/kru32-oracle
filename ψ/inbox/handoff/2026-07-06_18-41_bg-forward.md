## What We Did

- **ROOT CAUSE FIXED**: Browser crash during flash was `new_install_prompt_erase: true` in manifest. Full-chip erase triggers ESP32-S3 native USB (USB_SERIAL_JTAG) re-enumeration mid-transfer → hits Chromium 149 bug in Chrome_IOThread (EXC_BREAKPOINT / SIGTRAP, org.libusb.device-hotplug) → Chrome + Comet (same engine) both crash. **FIX**: `new_install_prompt_erase: false` on ALL manifests. Verified: no version mismatch (esp-web-tools 9.4.3 and 10.2.1 both work with erase:false).

- **esp-web-tools@10.2.1**: Upgraded to world latest. Self-hosted VENDORED copy was broken (missing dynamic-import chunks) → flash went silent. Now using unpkg full bundle (verified working in isolation test vs workshop-04-esp32-wasm flasher which never crashed).
