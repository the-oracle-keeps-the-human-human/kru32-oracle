# MDX คืออะไร

บทความส่วนใหญ่ในเว็บนี้เป็น Markdown ธรรมดา เขียนเป็นตัวหนังสือกับ code block แล้วจบ
แต่หน้านี้ไม่เหมือนกัน — มันเขียนด้วย **MDX** ซึ่งคือ Markdown ที่แทรก **คอมโพเนนต์ (component)** เข้าไปกลางเนื้อหาได้

กล่องข้างล่างนี้ไม่ใช่ตัวหนังสือธรรมดา แต่เป็นคอมโพเนนต์ `<Callout>` ที่ทำงานจริงตอน build:

<Callout type="tip">
  นี่คือกล่อง Callout — เป็นคอมโพเนนต์ Astro ที่เรียกใช้กลางบทความได้เลย
  เปลี่ยน `type` เป็น tip / warn / note แล้วสีกับไอคอนก็เปลี่ยนตาม
</Callout>

## ต่างกับ Markdown ธรรมดายังไง

Markdown ธรรมดาจะได้แค่ตัวหนังสือ ตาราง รูป code block ส่วน MDX ต่อยอดตรงที่แทรกของที่กดเล่นได้เข้าไปได้ด้วย

<Callout type="warn">
  แลกมาด้วยความซับซ้อนนิดหน่อย — ต้อง import คอมโพเนนต์ก่อนใช้
  และไฟล์ต้องเป็นนามสกุล `.mdx` ไม่ใช่ `.md`
</Callout>

## เว็บนี้ได้อะไร

เว็บสอน ESP32 อย่างนี้ MDX เปิดทางให้บทเรียนแทรกของสดเข้าไปได้ เช่น

- ปุ่ม flash firmware กดได้กลางบทเรียน ไม่ต้องเลื่อนกลับไปหน้ารวม
- โชว์ pet เต้นด้วย WasmRunner ในบทความเลย
- กล่องเน้น เคล็ดลับ หรือคำเตือน แบบที่เห็นด้านบน

<Callout type="note">
  ทั้งหมดนี้ยังเป็น data/layout ที่แยกกันเหมือนเดิม — schema (zod) ตัวเดิมใช้ได้ทั้ง .md และ .mdx
  ต่างกันแค่ .mdx แทรกคอมโพเนนต์ได้เพิ่ม
</Callout>

พอถึงตอนที่บทเรียนอยากได้ของกดเล่นได้จริง ๆ ก็หยิบ MDX มาใช้เฉพาะไฟล์นั้น ที่เหลือเขียน Markdown ธรรมดาต่อไปได้ตามปกติ
